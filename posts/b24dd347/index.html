使用 Hexo & Git 搭建个人博客（以及踩过的那些坑）

置顶 | 发表于 2020-02-16 更新于 2020-02-24 阅读次数：
这个博客是我使用 Hexo 和 Git 搭建的，由于没有代码基础，学习过程中踩了很多的坑，于是我打算在我的第一篇博客中，将这些坑一一记录，以备今后不时之需。

写在前面

这是我第一次搭建个人博客，这也是我的第一篇博客文章，实在想不到些什么好，于是干脆将这次搭建博客的全过程记录并整理，供自己不时参阅，也和大家一起沟通交流。

我自己的工作平台是 MacOS，从教程的反馈来看，Mac 与 Windows 平台的操作指令基本一致，仅部分界面样式有少许差别，故以下内容应该是通用的。

第一个坑：git clone 速度过慢

这里出现了第一个坑，当输入指令，出现 Cloning into 'Users/user_name/blog'... 字样后，下载速度过慢，会出现超时的失败报错；
如果运气好这一步下载成功没有报错，在后面 clone 主题资源的时候还是会遇到同样的问题，所以干脆在这里彻底解决。

经过搜索，网上普遍提供了两种解决方法：

  1. 使用系统代理工具对 git 请求进行中转 socks5://127.0.0.1:1080
  2. 在 hosts 文件中对 github.global.ssl.fastly.Net 和 github.com 这个域名绑定映射

由于方法 1 需要翻墙工具，这里暂时推荐零成本的方法 2：

  1. 访问网址 https://www.ipaddress.com/ 查询 github.global.ssl.fastly.net 和 github.com 域名的真实 IP ，截至 2020/02/23 23:48 查询结果见下图：
    [图片]
  2. 查询可知两者 IP 分别为 199.232.5.194 和 192.30.253.112
  3. 找到隐藏文件 /etc/hosts，Mac 系统在 Finder 中快速切换显示/隐藏文件的快捷键为 ⌘⇧.(Command + Shift + .)
  4. 使用 VS Code（在 1.2 安装的 Visual Studio Code 简称）打开 hosts 文件，将如下内容复制到文件最后并保存：

第二个坑：Markdown语法

恭喜，我们终于进入了「写」博客的巨坑，这里说难也不难，说容易也要学相当一段时间才能顺手。

这个话题如果要展开讲述的话，应该可以另写好几篇文章，所以这里仅简单罗列一下工具和需要学习的部分，以后有机会再另开新坑。

第五步：主题配置

5.1 下载主题

第三个坑：主题生态

我第一次更换主题时，按照视频教程选用了极简的 yilia，发现这个主题太过「极简」了，作者可能已经弃坑，最后一次更新已经是两年前，很多功能老旧，不太能满足我的配置需求；

然后又发现了 yilia-plus，在 yilia 的基础上增加了很多有意思的插件，但是作者夹带私货太多，在主题目录下的 _config.yml 可读性差，并非所有魔改项均可配置，遂舍弃；

最后换到了目前正在使用的主题：NexT
我经过一天的设置适应后，认为这个主题有以下优点：

  1. 同样是极简风，有多种布局可供切换；
  2. 插件种类丰富，使用说明文档完善，_config.yml 可配置性好；
  3. 多人维护，更新及时，截至我使用这个主题时，最新的 7.7.1 版本为 20 年 2 月刚刚更新。

这里再强烈安利一个文艺小清新风的博客主题：inside
[图片]
我个人非常喜欢，但是和我自己的风格不搭，所以没有尝试，如果有喜欢的小伙伴可以试一下这个主题。
